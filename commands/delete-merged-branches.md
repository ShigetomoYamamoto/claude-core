# マージ済みブランチ削除ルール

## 削除対象

以下の条件を**すべて**満たすブランチを削除対象とする：

1. **ローカルブランチである**（リモートブランチは対象外）
2. **developにマージ済みである**（`git branch --merged` で表示される）
3. **保護ブランチではない**（develop, main, master は削除しない）
4. **現在チェックアウト中のブランチではない**

## 削除対象の作業ブランチプレフィックス

| プレフィックス | 説明 |
|---------------|------|
| `feature/*` | 新機能追加ブランチ |
| `update/*` | 機能変更ブランチ |
| `fix/*` | バグ修正ブランチ |
| `remove/*` | 削除作業ブランチ |
| `refactor/*` | リファクタリングブランチ |
| `hotfix/*` | 緊急修正ブランチ |

## 保護ブランチ（削除禁止）

以下のブランチは削除対象から**常に除外**する：

- `develop`
- `main`
- `master`
- `staging`
- `production`

## 実行手順

1. マージ済みブランチの一覧を取得（`git branch --merged`）
2. 保護ブランチと現在のブランチを除外
3. 削除対象ブランチを表示
4. 各ブランチを `git branch -d` で削除
5. 削除結果を報告

## 出力例

```
🔍 マージ済みブランチを確認中...

削除対象ブランチ:
- feature/employee_csv_export_20251201
- fix/attendance_display_20251202
- update/salary_logic_20251203

🗑️ ブランチを削除中...
✅ Deleted: feature/employee_csv_export_20251201
✅ Deleted: fix/attendance_display_20251202
✅ Deleted: update/salary_logic_20251203

完了: 3件のブランチを削除しました
```

## 注意事項

⚠️ **削除されるのはローカルブランチのみ**です。リモートブランチは影響を受けません。

⚠️ **削除前に未プッシュのコミットがないか確認**してください。マージ済みであっても、ローカルにのみ存在するコミットがある場合は警告が表示されます。

⚠️ **削除対象が0件の場合**は「削除対象のブランチはありません」と表示して終了します。
