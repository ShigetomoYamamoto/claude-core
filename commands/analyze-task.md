# タスク分析

**task-analyst エージェント** を起動して、ユーザーから渡されたタスクや課題を実装可能な単位に整理します。

## このコマンドが行うこと

1. **タスクの取得** — GitHub Issue・PR コメント・自由形式テキストを読む
2. **分類** — バグ修正 / 機能追加 / リファクタ / チョア / 調査
3. **重要情報の抽出** — 現状・期待動作・再現手順・受け入れ基準
4. **曖昧点の特定** — 不明確な部分をリストアップ
5. **サブタスクへの分解** — 大きなタスクを 1-4 時間単位に分割
6. **ユーザーの承認待ち**

## いつ使うか

- サポートモードの起点として（タスク → PR）
- GitHub Issue を渡されたとき
- バグ報告・要望・改善依頼を受けたとき

## 全自動モード（`/requirements`）との違い

| コマンド | 想定入力 | 出力 |
|---|---|---|
| `/requirements` | 曖昧な要望・新機能アイデア | 機能要件・非機能要件 |
| `/analyze-task` | 具体的なタスク・Issue・バグ | 実装可能な単位への分解 |

## 次のステップ

タスク分析の承認後：

- `/plan` — 実装計画（ファイルパス・手順の段取り）
- `/tdd` — テスト駆動開発で実装

## 関連エージェント

`~/.claude/agents/task-analyst.md` を起動します。

## 注意事項

⚠️ **タスク分析をユーザーが承認するまで実装には進みません。**
