# 要件分析

**requirements-analyst エージェント** を起動して、ユーザーの自由形式の要望を実装可能な要件に変換します。

## このコマンドが行うこと

1. 要望を言い換えて確認
2. 機能要件（ユーザーストーリー + 受け入れ基準）の作成
3. 非機能要件（パフォーマンス・セキュリティ・スケーラビリティ等）の特定
4. スコープ境界（in / out / 将来）の定義
5. リスクと未解決の質問のリストアップ
6. **ユーザーの承認待ち**

## いつ使うか

- 全自動モードの起点として（要件 → 設計 → 実装 → デプロイ）
- 「〇〇を作りたい」「△△機能が欲しい」などの自由形式の要望を受けたとき
- 設計（`/design`）に進む前に要件を確定させたいとき

## 次のステップ

要件確定後：

- `/design` — システム設計（アーキテクチャ・DB スキーマ・API コントラクト）
- `/plan` — 実装計画（既存設計の範囲内で進める場合）

## 関連エージェント

`~/.claude/agents/requirements-analyst.md` を起動します。

## 注意事項

⚠️ **要件をユーザーが承認するまで設計・実装には進みません。**
