# 3-line-contract — 目的・出力・成功条件を3行に固める

実装や VISION 作成の前に、**たった3行**でタスクの輪郭を固めるスキル。曖昧なまま進むと往復が増えるので、ここで「何を・何が出れば・どうなれば成功か」を言葉にする。loop-engineering の VISION(STEP 2)の前段にあたる。

## 3行の型

```
目的(Why):    このタスクは何のためか(1行)
出力(What):   何が成果物として出るか(1行・観測可能に)
成功条件(Done): どうなったら完了と言えるか(1行・できれば機械検証可能に)
```

## 書き方

1. **目的(Why)** — 「誰が・何に困っていて・何を可能にしたいか」。
   例: 「保存ボタンで入力が消えるバグで、ユーザーが入力をやり直す手間をなくす」
2. **出力(What)** — 外から見える成果物。コード・ファイル・画面など具体物で。
   例: 「保存後も入力が残るフォーム」
3. **成功条件(Done)** — ○× が付く形で。loop-engineering の VISION 述語の種になる。
   例: 「保存 → リロード後も入力値が残る(テストで検証)」

## コツ

- **1行に収まらない = スコープが大きすぎ**。分割を検討する。
- 成功条件は **機械検証可能**を目指す(`~/.claude/rules/loop-safety.md` の Precondition「一文で言え機械検証可能な done-condition」と同じ思想)。測れないものは `[AI]` 判定になる旨を添える。
- 答えられない項目があれば、似た例を2〜3個こちらから出して利用者に選ばせる(ゼロから言わせない)。

## 出力フォーマット

```
## 3行コントラクト
- 目的:    …
- 出力:    …
- 成功条件: …
```

これを持って loop-engineering の VISION(STEP 2)に進むと、合格条件がぶれない。
