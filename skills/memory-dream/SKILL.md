# memory dream（記憶の整理／consolidation）

本Skillは、エージェントが各セッションで蓄積した記憶階層を定期的に再編し、重複・矛盾・陳腐化したエントリを排除して「動く知見」へと整える自律タスクの実行手順書です。

## 1. 探索・対象ファイルの洗い出し（Mine）
以下のグローバルパスおよび個別プロジェクトフォルダから、対象となる設定ファイルや記憶ファイルを探索・リストアップしてください。

- **グローバル領域**: `~/.claude/` や、共有記憶リポジトリ（例: `agents-share`）の配下
- **プロジェクトローカル領域**: 開発用ディレクトリ（例: `~/projects/` や `~/workspace/`）の配下にある、個別リポジトリの `.claude/`, `.claudecode/` 配下の Markdown ファイル、または `MEMORY.md` などの記憶ファイル
- **探索コマンド例**:
  ```bash
  # 各プロジェクト内の記憶ファイルを検索する
  find ~/projects -type f -name "MEMORY.md" -o -path "*/.claude*/*.md"
  ```

## 2. 記憶の統合と更新（Consolidate）
リストアップされたすべてのファイルを読み込み、直近の更新履歴や指摘から重要な知見を抽出して、既存の記憶ファイルへマージしてください。
- **相対日付の変換**: 「昨日」「先週」「数日前」などの曖昧な表現は、すべて実行時点を基準とした【絶対日付（例: 2026年7月3日）】へ厳密に変換してください。
- **矛盾の解消**: 古い記述や廃止された手順は、最新の事実や整合性の取れる記述で完全に置き換えてください。
- **参照の現存確認**: 存在しないファイル、削除済みの関数、廃止された設定フラグなどを指す記述は、現存確認を必ず行い、残存していなければ除去してください。

## 3. 重複排除と階層ルールの遵守（Dedup & Resolve）
グローバル（上位）と個別プロジェクト（下位）の間で重複するルールがある場合、重複排除を行います。
- **最重要原則**: グローバルルール（例: `AGENTS.md`）が定義する「世界のルール」は、下位のプロジェクト固有記憶や個別Skill手順で再掲してはなりません。
- **下位のクリーンアップ**: 下位レイヤ側に存在する重複したルールは黙って削除し、そのプロジェクトやレイヤ特有の「固有の知見」のみを簡潔に残してください。

## 4. 履歴の徹底排除と剪定（Prune & Index）
成果ファイル（`MEMORY.md` / 各プロジェクト設定）に「メタな経緯」や「履歴」を残さないでください。
- **除外するもの**:
  - 重複回避のための注記（「これは上位ルールが定めるためここでは記述しない」等の言い訳）
  - 経緯、失敗談（「同じミスを3回繰り返した」「以前〇〇で失敗した」等の技術的因果以外の履歴）
  - 学習日、セッションID、誰から指摘されたかなどの個人情報やメタデータ
- **残すもの**: 現行で「正しく動く手順、知見、技術的因果（例: AだとBが壊れるためCする）」のみ。
- **索引の同期**: `MEMORY.md` 内の「notes 一覧」セクションを、以下のコマンドを用いて再生成・同期してください。
  ```bash
  find notes -type f -name '*.md' | sort
  ```

## 5. チェックリスト（実行前確認）
作業完了前に、必ず以下のチェックリストをすべて満たしているかセルフレビューを行ってください。
- [ ] 相対日付をすべて絶対日付へ変換した
- [ ] 上位が定めるルールを下位から削った（重複回避の注記自体も残していない）
- [ ] 成果ファイルから経緯・履歴（Why / 失敗談 / 再発回数 / 学習日 / セッション ID）を除いた
- [ ] 矛盾を最新値で解決した
- [ ] 存在しないファイル/シンボルへの参照を除去 or 現存確認した
- [ ] 成果ファイルをコミット（または別ブランチにコミット）し、ユーザーがレビューできる状態にした
