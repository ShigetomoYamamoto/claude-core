# ADR-021: 言語ポリシー改定 — rules は英語基調＋日本語の運用注記を許容する

**ステータス**: accepted

**日付**: 2026-07-04

**部分的に置き換える対象**: [ADR-003](./003-language-policy.md)（rules を英語のみとしていた点。agents=英語 / commands=日本語 は不変）

## コンテキスト

[ADR-003](./003-language-policy.md) は「rules・agents は英語、commands は日本語」とした。しかし運用の実態として、rules には日本語の節が恒常的に増えた（`rules/autorun-flow.md` の Remote CI 節、`rules/git-workflow.md`「コマンドの所在」、`rules/agents.md` の公式委譲・物理層注記、`rules/security.md` の防御層表など）。これらは「モデルへの指示」であると同時に「運用者（日本語話者）が transcript で読み・監査する規範」でもあり、日本語の方が目的に適う箇所だった。2026-07-04 のナレッジベース棚卸しで、ポリシーと実態の乖離として顕在化した。

## 検討した選択肢

1. **rules を全て英語へ書き戻す** — ポリシー一貫性は回復するが、運用者向け注記の可読性を失い、書き戻し工数と回帰リスクが大きい。実態はまた日本語に戻る見込みが高い。
2. **rules を全て日本語化** — モデル指示精度に関する ADR-003 の懸念が残り、既存英語部分の書き換え工数も大きい。
3. **実態追認（採用）** — rules は「英語基調。ただし主読者が運用者である節（安全規範の監査注記・委譲手順・停止条件の説明）は日本語可」。agents は英語、commands は日本語のまま。

## 決定

**選択肢3を採用する。** 判断基準は「その文章の主読者は誰か」: モデルへの動作指示が主なら英語、運用者の理解・監査が主なら日本語可。同一ファイル内の混在を許容する。agents（英語）と commands（日本語）は ADR-003 のまま。

## 結果

### Positive

- ポリシーが実態と一致し、「違反の常態化」が解消する。
- 安全規範のうち人間が監査する部分の可読性を言語面で最適化できる。

### Negative

- 1ファイル内の言語混在は残る（見た目の一貫性は下がる）。
- 「主読者がどちらか」の判断は書き手依存で、揺れが残りうる。
