# ADR-006: hook の例外処理ポリシー

**ステータス**: accepted

**日付**: 2026-05-21

> **注記（2026-07-06）**: 「1 ファイル 100 行以下」の制約は実装の成長により超過が常態化した（`mass-delete-blocker.py` 121行・`git-destructive-blocker.py` 103行）。本制約は上限規則ではなく**目安**として運用する（単一責務・ネットワーク通信禁止は不変）。2026-07-06 の棚卸しによる実態追認。

## コンテキスト

hook（Python スクリプト）は Claude Code の動作に介入する。hook 自体に予期せぬ例外（バグ・環境差・ファイル読み込み失敗）が発生したとき、Claude の動作を止めるべきか、無視すべきかの方針が必要。

## 検討した選択肢

1. **すべて例外で Claude を止める（exit 1）** — エラーを確実に検知できるが、hook のバグで開発全体が止まる
2. **すべて exit 0 で無視** — 動作の堅牢性は高いが、意図的にブロックしたい hook（doc-blocker）が機能しない
3. **役割別に方針を変える** — 検知系は exit 0、意図的ブロックは exit 2

## 決定

**役割別に方針を変える:**

| 種別 | 例 | 終了コード |
|---|---|---|
| 意図的なブロック | `doc-blocker.py` | exit 2（Claude を止める） |
| 検知系（警告のみ） | `secret-detection.py` | exit 0 + メッセージ出力 |
| 予期せぬ例外 | すべての hook | exit 0（try/except で握りつぶす） |

### 実装例（Python）

```python
try:
    # メインロジック
    ...
except Exception:
    sys.exit(0)
```

### 制約

- hook はネットワーク通信禁止（ローカル処理のみ）
- 1 ファイル 100 行以下
- 単一責務

## 結果

### Positive

- hook のバグで開発全体が止まることがない
- 意図的なブロック（不要ドキュメント生成防止など）は確実に機能する
- 検知系は警告だけ出して Claude を続行させる
- 役割の境界が終了コードで明確に分かれる

### Negative

- hook の例外がログに残らない可能性（exit 0 で握りつぶされる）
- 「ブロックすべき」か「警告のみ」かの判断が hook ごとに必要
